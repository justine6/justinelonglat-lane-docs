JLT Platform — Access Model

The JLT Platform Access Model defines how users, contributors, subscribers, and administrators interact with platform resources, APIs, documentation, automation assets, and downloadable toolkits.

1. Identity

Every request to the platform is associated with an identity. Identity may come from Microsoft 365 (guest contributors), platform login, or internal admin accounts.

  • Public visitor
  • Guest contributor
  • Basic member
  • Toolkit subscriber
  • Platform subscriber
  • Advisory client
  • Admin
  • Owner

2. Subscription Tiers

Subscription tiers determine what protected resources a user can access. Access is not granted by login alone, but by entitlement tied to a subscription tier.

  • None
  • Starter
  • Toolkit
  • Platform
  • Advisory
  • Internal

3. Resource Groups

All platform resources are grouped into resource groups. Access control policies are applied at the resource group level.

  • docs.public
  • docs.private
  • comments
  • contributor.protocols
  • downloads.starter
  • downloads.premium
  • toolkit.preview
  • toolkit.full
  • runbooks.public
  • runbooks.private
  • automation.templates.basic
  • automation.templates.premium
  • api.platform.read
  • api.platform.execute
  • api.admin
  • billing.customer
  • billing.admin

4. Entitlements

Entitlements are explicit permissions required to access specific routes or downloads. Even if a user is authenticated, access is denied unless the required entitlement is present.

Examples:

  • docs.read.public
  • comment.submit
  • contributor.protocol.read
  • downloads.starter
  • downloads.premium
  • runbooks.read.private
  • api.platform.read
  • api.platform.execute
  • billing.admin

5. Access Control Rules

  • All protected routes require entitlement checks.
  • All premium downloads are served through protected API routes.
  • Direct public access to premium assets is not allowed.
  • Default access policy is deny unless explicitly allowed.
  • Access decisions are evaluated per request.

6. Access Flow

User → Authentication → Role → Subscription Tier → Entitlements
     → Resource Group Policy → Access Decision (Allow / Deny)

7. Platform Boundary Model

User Type Access Level
Public Visitor Public docs and blog
Guest Contributor Contributor protocols and collaboration areas
Subscriber Premium downloads, toolkit, private runbooks
Advisory Client Client-specific resources and deliverables
Admin Platform management and billing

8. Security Principle

The JLT Platform follows a deny-by-default access model. Access is granted only when identity, subscription, entitlements, and resource policy all allow the requested action.
