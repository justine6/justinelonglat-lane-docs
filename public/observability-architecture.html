Platform Observability Architecture

JLT-Lane builds observability directly into the engineering platform behind modern applications. This page documents the monitoring flow used in the sandbox platform environment.

Overview

In this architecture, a Node.js authentication service exposes operational metrics through a /metrics endpoint. Prometheus scrapes those metrics at regular intervals, stores them as time-series data, and Grafana visualizes service behavior through dashboards.

All components run inside a Docker-based environment, making the platform portable, reproducible, and easy to operate.

Core Components

Node.js Auth Service

The service exposes Prometheus-compatible metrics that describe internal runtime behavior and application activity. 

process_cpu_seconds_total
process_resident_memory_bytes
nodejs_active_handles
auth_service_http_requests_total

Prometheus

Prometheus scrapes the metrics endpoint and stores the results as queryable time-series data. This forms the measurement layer of the observability stack. 

scrape_configs:
  - job_name: "auth-service"
    metrics_path: /metrics
    static_configs:
      - targets: ["auth-service:3002"]

Grafana

Grafana connects to Prometheus and renders dashboard panels for service health, request throughput, CPU usage, memory consumption, and runtime pressure indicators.

Example PromQL Queries

rate(auth_service_http_requests_total[5m])
rate(process_cpu_seconds_total[1m])
process_resident_memory_bytes
nodejs_active_handles

Why This Matters

Modern systems should not just run. They should be able to explain their own behavior. Observability helps teams detect issues early, troubleshoot incidents faster, and operate services with more confidence.

At JLT-Lane, this is part of the platform foundation itself rather than an afterthought.

JLT-Lane Platform View

JLT-Lane builds the engineering platform behind your applications — the infrastructure, automation, monitoring, and security systems that allow your services to run reliably and scale as your organization grows.

This observability architecture is one practical example of that philosophy in action.
